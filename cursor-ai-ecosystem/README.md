# Cursor AI Ecosystem - Экосистема ИИ

## Описание

Cursor AI Ecosystem - это амбициозный проект по созданию самоорганизующейся экосистемы искусственного интеллекта, основанной на концепциях:
- **Нуклеотидной памяти** (62 млн ячеек по 256 байт)
- **Вокселей-организмов** (9 КБ каждый) с эмоциями, мыслями и памятью
- **Системы "кайфа"** на основе энтропии
- **Автоматического поиска концептов** через DuckDuckGo
- **Паттернов освещения** для визуализации

## Архитектура

### Структуры данных

1. **Nucleotide** (256 байт) - базовая ячейка памяти с:
   - Базой (A, T, G, C)
   - Эпигенетическими метками
   - Квантовым шумом
   - Состоянием гистонов
   - Семантическим вектором (512 значений)

2. **Voxel** (9 КБ) - микро-организм с блоками:
   - 512 Б - метаданные
   - 1.5 КБ - сенсоры
   - 1 КБ - физика
   - 2 КБ - мысли
   - 2 КБ - эмоции
   - 2 КБ - память

3. **LightPattern** (1 КБ) - паттерн освещения с:
   - ID паттерна
   - Прямым освещением (32 источника)
   - Непрямым освещением
   - Коэффициентами сферических гармоник
   - Свойствами материала

## Использование

### Основной режим
```bash
python main.py
```

### Параметры запуска
- `--nucleotides N` - количество нуклеотидов (по умолчанию 1000000)
- `--voxels N` - количество вокселей (по умолчанию 1000)
- `--visualize` - включить визуализацию
- `--no-search` - отключить поиск концептов

## Возможности

### 1. Нуклеотидная память
- 62 миллиона нуклеотидов (опционально, для полной системы)
- Эпигенетические метки
- Квантовый шум для случайности
- Семантические векторы

### 2. Воксели-организмы
- Внутренние потоки между блоками
- Система эмоций и мыслей
- Адаптация на основе "кайфа"
- Память о прошлом опыте

### 3. Система "кайфа"
- Вычисление через |dS/dt| (производная энтропии)
- Усиление связей при высоком кайфе
- Адаптация поведения

### 4. Поиск концептов
- Автоматический поиск каждые 19 минут
- Интеграция новых терминов
- Расширение базы знаний

### 5. Паттерны освещения
- База паттернов освещения
- Поиск похожих паттернов
- Смешивание для итогового освещения

### 6. Визуализация
- Отображение состояния вокселей
- График энтропии и "кайфа"
- Визуализация памяти

## Архитектурные особенности

- **Асинхронность**: использование `asyncio` для параллельного выполнения задач
- **Оптимизация**: NumPy для обработки массивов
- **Модульность**: независимые компоненты
- **Расширяемость**: система плагинов (планируется)

## Технические детали

### Обновление нуклеотидов
Каждые 16 мс (60 FPS):
- Обновление квантового шума
- Корректировка эпигенетических меток
- Сдвиг семантических векторов

### Обновление вокселей
- Перекачка данных между блоками
- Вычисление "кайфа"
- Адаптация связей

### Планировщик задач
- Ticker: 60 FPS обновление
- Concept Fetcher: каждые 19 минут
- Async/await для неблокирующего выполнения
